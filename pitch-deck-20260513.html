The Solution

Prophet: the first prediction market
where everything runs itself.

AMM Binary YES/NO markets

Every market has YES and NO shares priced $0–$1 by a protocol-owned AMM. No human LPs needed. Liquidity is seeded from a protocol pool and managed by an autonomous agent.

ORACLE AI resolution via 0G Compute

At deadline, the oracle agent calls 0G Compute, reasons about the outcome, writes the full chain-of-thought to 0G Storage, and posts the verdict on-chain. No human vote. Confidence threshold enforced.

SEALED Private positions via TEE

Bet commitments are NaCl-encrypted before submission. PositionVault stores only the ciphertext. Direction and amount are revealed simultaneously at resolution — zero front-running window.

AGENTS Autonomous operations 24/7

Market maker agent allocates liquidity, monitors AMM reserves, queues txs to avoid nonce collisions, and returns settled funds. Oracle agent scans for expired markets and drives them to resolution.

Speaker Notes
Four pillars — AMM, oracle, privacy, agents. The key insight: none of these four things works without the others. An AMM without liquidity dies. An oracle without accountability is just a black box. Privacy without simultaneous reveal is theatre. Agents without reliable compute and storage can't operate autonomously. All four together, on 0G, is the product.

Architecture

Five contracts. Two autonomous agents. Three 0G modules.
One full market lifecycle.

  1. Create — User submits question → 0G Compute validates it → ProphetFactory deploys MarketContract → metadata written to 0G Storage
  2. Seed — Market maker agent detects new market → reads LiquidityPool → allocates USDT to AMM reserves → YES/NO shares are immediately tradable
  3. Trade — Users buy/sell YES and NO shares → AMM price moves with slippage → sealed commitments stored in PositionVault (encrypted, direction hidden)
  4. Resolve — Deadline passes → oracle agent calls 0G Compute → reasoning written to 0G Storage → verdict posted on-chain → 24h challenge window → finalizeResolution()
  5. Settle — Positions revealed simultaneously → winning shares redeem at $1.00 → losing shares at $0.00 → pool liquidity returned → fees distributed
Speaker Notes
Walk through each step as a user journey. Emphasise step 4 — the oracle flow is where all three 0G modules interact simultaneously: 0G Compute for inference, 0G Storage for the reasoning record, 0G Chain for the verdict transaction. That's the core of the product and the core of the 0G integration.
